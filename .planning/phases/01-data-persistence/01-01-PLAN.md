## Goal

Create `src-tauri/src/db/queries.rs` with typed CRUD functions that wrap raw SQL calls into strongly-typed Rust operations. This is the foundation layer that all Tauri commands will call.

## must_haves

- All 11 functions compile and return correct types
- `anyhow::Result` used for all fallible returns
- `&tauri_plugin_sql::Db` as first parameter on every function
- `db.execute()` for writes, `db.select()` for reads
- `serde_json::from_value` for parsing query results
- `uuid::Uuid::new_v4()` for ID generation
- `chrono::Utc::now()` for timestamp generation
- CASCADE behavior on `delete_conversation` verified against migration SQL
- `pub mod queries;` added to `db/mod.rs`

### Task 2: Implement conversation CRUD functions

<read_first>
src-tauri/src/db/models.rs
src-tauri/migrations/001_init.sql
</read_first>

<acceptance_criteria>
- `list_conversations` returns `Result<Vec<Conversation>>` sorted by `created_at DESC`
- `get_conversation` returns `Result<Option<Conversation>>`
- `create_conversation` generates UUID for id, sets `created_at` and `updated_at` to current UTC, inserts row, returns the created `Conversation`
- `update_conversation_title` executes UPDATE with new title and current `updated_at`
- `delete_conversation` executes DELETE by id (migration has ON DELETE CASCADE for messages)
- All functions use parameterized queries (no string interpolation)
</acceptance_criteria>

<action>
Implement these 5 async functions:

1. `list_conversations(db: &Db)` — SELECT * FROM conversations ORDER BY created_at DESC. Use `db.select()` and parse rows into `Vec<Conversation>`.

2. `get_conversation(db: &Db, id: &str)` — SELECT * FROM conversations WHERE id = ?. Return `Ok(None)` if empty result.

3. `create_conversation(db: &Db, title: &str)` — Generate `Uuid::new_v4()`, capture `Utc::now().to_rfc3339()` for both timestamps. INSERT INTO conversations (id, title, created_at, updated_at). Return the constructed `Conversation` struct.

4. `update_conversation_title(db: &Db, id: &str, title: &str)` — UPDATE conversations SET title = ?, updated_at = ? WHERE id = ?.

5. `delete_conversation(db: &Db, id: &str)` — DELETE FROM conversations WHERE id = ?. Rely on ON DELETE CASCADE in migration for messages.
</action>

</acceptance_criteria>

<action>
Implement 2 async functions:

1. `get_active_conversation_id(db: &Db)` — SELECT value FROM app_settings WHERE key = 'active_conversation_id'. Parse single scalar value. Return None on empty result.

2. `set_active_conversation_id(db: &Db, id: &str)` — Try INSERT INTO app_settings (key, value) VALUES ('active_conversation_id', ?). On conflict (key already exists), do UPDATE SET value = ?. This is an upsert pattern — use INSERT OR REPLACE or ON CONFLICT depending on SQLite dialect supported by the plugin.
