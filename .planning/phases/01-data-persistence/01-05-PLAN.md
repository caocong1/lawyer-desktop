## Goal

Implement automatic conversation title generation from the first message content. Ensure active conversation ID is always persisted when switching or creating conversations. Complete the data persistence loop so the app state fully survives restarts.

## must_haves

- Title auto-generated from first message: first 20 chars, trimmed
- Title generation triggers only when current title is "цЦ░ф╝ЪшпЭ" (not on subsequent messages)
- `updateConversationTitle` API wrapper added
- `update_conversation_title` Tauri command added and registered
- `switchConversation` and `createConversation` always persist active conversation ID
- Local conversations signal updated immediately after title change (optimistic update)
- `bunx tsc --noEmit` passes
- `cargo check` passes

```rust
#[tauri::command]
pub async fn update_conversation_title(
    db: State<'_, tauri_plugin_sql::Db>,
    conversation_id: String,
    title: String,
) -> Result<(), String> {
    db::queries::update_conversation_title(&db, &conversation_id, &title)
        .await
        .map_err(|e| e.to_string())
}
```
