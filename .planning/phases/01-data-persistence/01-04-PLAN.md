## Goal

Wire `App.tsx` to hydrate application state from the database on launch. Replace hardcoded sidebar data with live conversation list from DB. Add loading overlay. Ensure Settings panel shows automatically when no provider is configured.

## must_haves

- App.tsx `onMount` calls `settings.restoreProvider()` first, then `conversation.loadConversations()`
- Loading overlay shows "加载中..." while initial data loads
- Sidebar renders `conversations()` signal (not hardcoded list)
- Sidebar has delete button per conversation calling `conversation.deleteConversation(id)`
- "新会话" placeholder shown for conversations with empty/default title
- Settings panel auto-shows when `isConfigured()` is false after restore
- Active conversation restored from DB if `activeConversationId()` is null on startup
- `bunx tsc --noEmit` passes after all changes

### Task 1: Add bootstrap logic to App.tsx

<read_first>
src/App.tsx
src/stores/conversation.ts
src/stores/settings.ts
</read_first>

<acceptance_criteria>
- `onMount` or initial `createEffect` executes bootstrap sequence
- Bootstrap order: (1) restoreProvider, (2) loadConversations, (3) restore active conversation, (4) hide loading
- `isLoading` signal created (local or from a store) — initially `true`
- After provider restored, if `!isConfigured()`, show Settings panel (set `showSettings` or equivalent signal to true)
- After conversations loaded, check `activeConversationId()` — if null, try to restore from DB or auto-select first conversation
- Loading overlay rendered conditionally via `<Show when={isLoading()}>` wrapping a centered "加载中..." element
- `isLoading` set to `false` after all bootstrap steps complete (in a `finally` block or after last await)
</acceptance_criteria>

<action>
Add to App.tsx:

1. Create `const [isLoading, setIsLoading] = createSignal(true);` before onMount.

2. In `onMount`, add async bootstrap:
```
const bootstrap = async () => {
  try {
    await settings.restoreProvider();
    if (!settings.isConfigured()) {
      // trigger showing settings panel
    }
    await conversation.loadConversations();
    // restore active conversation if needed
    const convs = conversation.conversations();
    if (!conversation.activeConversationId() && convs.length > 0) {
      await conversation.switchConversation(convs[0].id);
    }
  } catch (e) {
    console.error("启动加载失败:", e);
  } finally {
    setIsLoading(false);
  }
};
bootstrap();
```

3. Add loading overlay in JSX, wrapping main content or as a sibling:
```
<Show when={isLoading()}>
  <div class="loading-overlay">加载中...</div>
</Show>
```

4. Add CSS for loading overlay in App.css or inline styles.

Adjust exact signal names and component structure to match existing code.
</action>
