## Goal

Wire the DB query layer (from 01-01) into Tauri command functions. Add `State<'_, tauri_plugin_sql::Db>` parameter injection. Update existing commands to persist data, and add new commands for frontend data retrieval.

## must_haves

- All existing commands continue to work (backward compatible)
- `State<'_, tauri_plugin_sql::Db>` added to commands that need DB access
- New commands: `get_conversations`, `get_messages`, `delete_conversation`, `get_active_provider`, `set_active_conversation`
- Existing `create_conversation` persists to DB after creating struct
- Existing `send_message` saves both user and assistant messages to DB
- Existing `setup_provider` persists provider config to DB
- All new commands registered in `lib.rs` `generate_handler![]`
- Error boundary: `anyhow::Result` mapped to `Result<T, String>` via `.map_err(|e| e.to_string())`

### Task 3: Add new read/delete commands in chat.rs

<read_first>
src-tauri/src/commands/chat.rs
src-tauri/src/db/queries.rs
</read_first>

<acceptance_criteria>
- `get_conversations` command returns `Result<Vec<Conversation>, String>` by calling `queries::list_conversations`
- `get_messages` command takes `conversation_id: String` parameter, returns `Result<Vec<Message>, String>` by calling `queries::list_messages`
- `delete_conversation` command takes `conversation_id: String` parameter, calls `queries::delete_conversation`
- `delete_conversation` also checks if deleted conversation was the active one — if so, calls `queries::set_active_conversation_id(&db, "")` or clears it
- All three commands have `#[tauri::command]` attribute
- All three have `State<'_, tauri_plugin_sql::Db>` parameter
</acceptance_criteria>

<action>
Add three new `#[tauri::command]` async functions to `chat.rs`:

1. `pub async fn get_conversations(db: State<'_, tauri_plugin_sql::Db>) -> Result<Vec<db::models::Conversation>, String>` — calls `db::queries::list_conversations(&db).map_err(|e| e.to_string())`

2. `pub async fn get_messages(db: State<'_, tauri_plugin_sql::Db>, conversation_id: String) -> Result<Vec<db::models::Message>, String>` — calls `db::queries::list_messages(&db, &conversation_id).map_err(|e| e.to_string())`

3. `pub async fn delete_conversation(db: State<'_, tauri_plugin_sql::Db>, conversation_id: String) -> Result<(), String>` — calls `db::queries::delete_conversation(&db, &conversation_id).map_err(|e| e.to_string())?`. Then check active: call `db::queries::get_active_conversation_id(&db)`, if it equals the deleted id, call `db::queries::set_active_conversation_id(&db, "")`.
</action>
